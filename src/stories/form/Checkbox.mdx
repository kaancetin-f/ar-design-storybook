`Checkbox` is the core form component used to let users select one or more independent
options from a list, or toggle a single binary choice (e.g. "I agree to the terms and
conditions"). It comes with multiple color, variant, size, and border radius options, and
can be grouped together with `CheckboxGroup` to manage a related set of choices.

> **Note:** Use `Checkbox` when multiple options can be selected at the same time. If the
> user must choose only _one_ option from a list, use a **Radio Button** instead.

```typescript
import { Checkbox, CheckboxGroup } from "@harjs/react-ui";
```

> **Note:** All examples below are live — you can use Storybook's Controls panel to change
> props and instantly test the component's behavior.

## Variant

`Checkbox` offers several visual variants, each carrying a different level of emphasis.
Variants with a stronger fill or border stand out more clearly and work well when the
choice itself carries weight, while lighter, more outlined options blend into the
surrounding interface and suit denser lists where many checkboxes appear together. For
consistency in your design language, it's recommended to use a single variant within the
same form.

## Color

The component supports a wide range of color options so it can align with your brand
palette. Color here isn't just decorative — it can be used to convey meaning, such as
tying a checkbox's color to the category it represents in a filtering system, or to a
status it controls. Using semantic colors consistently across the project helps users
build a reliable mental model of what each color means.

<Canvas of={CheckboxStories.Color} />

## Disabled

Prevents any user interaction while visually indicating that the checkbox is currently
inactive — the field is dimmed and the cursor switches to `not-allowed`. This is useful for
conditional forms where a choice depends on another field being filled in first, or for
options that are temporarily unavailable. For accessibility, `disabled` checkboxes are also
automatically skipped by screen readers.

<Canvas of={CheckboxStories.Disabled} />

## Border Radius

Corner rounding can be customized across a wide range of options and is an important visual
variable that determines the overall "hard/soft" feel of your design system. Sharper,
square corners tend to read as more structured and align with denser, data-heavy
interfaces, while fully rounded corners feel softer and more approachable. For consistency
across the project, it's generally recommended to use a single border radius value across
all form controls.

<Canvas of={CheckboxStories.Radius} />

## Size

The component offers multiple size options that can be chosen based on context. Smaller
sizes are preferred for compact, dense lists where many options are shown at once, while
larger sizes improve touch-target area and visibility on mobile or in forms where the
choice needs to stand out.

<Canvas of={CheckboxStories.Size} />

## Validation

Displays feedback states such as error or success markers. When a validation rule fails —
for example, a required agreement checkbox left unchecked — the component visually adapts
to draw the user's attention to the correction that's needed before the form can proceed.
