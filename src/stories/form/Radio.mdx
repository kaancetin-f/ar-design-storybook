`Radio` is the core form component used when users must select a single option from a
predefined set. It ensures that only one choice can be active at a time within the same
group, automatically deselecting the previous option when a new one is chosen. It comes
with multiple color, variant, and size options, and can be grouped together with
`RadioGroup` to manage a related set of mutually exclusive choices.

> **Note:** Use `Radio` when the user must choose **only one** option from a list. If
> multiple options can be selected independently, use a **Checkbox** instead.

```typescript
import { Radio, RadioGroup } from "@harjs/react-ui";
```

> **Note:** All examples below are live — you can use Storybook's Controls panel to change
> props and instantly test the component's behavior.

## Variant

`Radio` offers several visual variants, each carrying a different level of emphasis.
Variants with a stronger fill or border stand out more clearly and help draw attention to
the selected option, while lighter, more outlined options blend into the surrounding
interface and suit denser lists where many options appear together. For consistency in
your design language, it's recommended to use a single variant within the same form.

## Color

The component supports a wide range of color options so it can align with your brand
palette. Color can be used to emphasize the selected state more clearly or to tie a group
of options to your application's broader theme. Using semantic colors consistently across
the project helps users build a reliable mental model of what each color means.

<Canvas of={RadioStories.Color} />

## Disabled

Prevents user interaction while clearly indicating that the option is currently
unavailable — the field is dimmed and the cursor switches to `not-allowed`. This is useful
for conditional forms where an option depends on another field being filled in first, or
for choices that are temporarily inaccessible. For accessibility, `disabled` options are
also automatically skipped by screen readers.

<Canvas of={RadioStories.Disabled} />

## Size

The component offers multiple size options that can be chosen based on context. Smaller
sizes are preferred for compact, dense lists where many options are shown at once, while
larger sizes improve touch-target area and visibility on mobile or in forms where the
decision needs to stand out.

<Canvas of={RadioStories.Size} />

## Validation

Displays validation feedback such as error or success states. This is useful for required
selections, where the component visually adapts to draw the user's attention to the
correction that's needed before the form can proceed.
